Overview

The basis.sequential package harbors single-threaded, high-level collections operations. Three broad categories organize the extensions in this package: general operations analyze or reduce collections to produce stand-alone results; strict operations eagerly act on collections to produce new, self-contained collections; and non-strict operations return derivative collections that lazily evaluate their elements.

The compiler dispatches operations based on the static type of the receiving collection. Static resolution enables macros to inline general and strict operations for Iterator and Container subtypes, with greater refinement for Index and Stack types. These macros produce compact, highly efficient bytecode, and permit the compiler to elide closure class generation. This lets concise, high-level operations perform like rote, low-level ones.

General Operations

The General class includes implicit conversions that add reductive operations to all collection types. To use these extensions, import the contents of the general value in the sequential package object as follows:

import basis.sequential.general._

Strict Operations

The Strict class extends General to include implicit conversions that add eagerly evaluated transformations to all collection types. To use these extensions, import the contents of the strict value in the sequential package object as follows:

import basis.sequential.strict._

Strict subsumes General, so don't import both strict._ and general._; that will cause ambiguities.

Non-Strict Operations

The NonStrict class extends General to include implicit conversions that add lazily evaluated transformations to all collection types. To use these extensions, import the contents of the nonstrict value in the sequential package object as follows:

import basis.sequential.nonstrict._

NonStrict subsumes General, so don't import both nonstrict._ and general._; that will lead to ambiguities.
